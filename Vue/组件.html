# 组件

# 单个根元素

每个组件必须只有一个根元素，模板的内容必须包裹在一个父元素内

# data必须是一个函数

一个组件的 data 选项必须是一个函数，因此每个实例可以维护一份被返回对象的独立的拷贝。如果data是一个对象，当页面中有多个相同组件实例时，它们之间会相互影响。

考虑下js的闭包

# 组件间通信

# 父组件向子组件通信

  • 通过 Prop 向子组件传递数据(单项数据流)

    我们能够在组件实例中访问这个值，就像访问 data 中的值一样

# 子组件向父组件通信

  • 监听子组件事件$emit

    Vue 实例提供了一个自定义事件的系统，父级组件可以像处理 native DOM 事件一样通过 v-on 监听子组件实例的任意事件，同时子组件可以通过调用内建的 $emit 方法 并传入事件名称来触发一个事件

    // 自定义事件${myEvent}
<my-component v-on:${myEvent}>="eventHandler"></my-component>

// 子组件触发
this.$emit('${myEvent}'[,data]); // data参数可选，可以作为子组件向父组件传递数据

# 依赖注入

多个子组件都需要用到父组件方法时，通过依赖注入更佳便利 父组件中：

provide: function () {
  return {
    getMap: this.getMap
  }
}

子组件中：

inject: ['getMap']

赖注入还是有负面影响的。它将你的应用以目前的组件组织方式耦合了起来，使重构变得更加困难

# 插槽slot

Vue 实现了一套内容分发的 API，将 <slot> 元素作为承载分发内容的出口。

// 这样调用组件
<navigation-link url="/profile">
  Your Profile
</navigation-link>

// <navigation-link> 的模板
<a v-bind:href="url" class="nav-link">
  <slot></slot>
</a>

当组件渲染的时候，<slot></slot> 将会被替换为“Your Profile”。

插槽内可以包含任何模板代码，包括 HTML和组件

子组件内slot可以添加默认值，如果父组件没有传入显示默认值，传入内容会替换默认值

# 编译作用域

插槽跟它所在的模板具有相同的作用域,父级模板里的所有内容都是在父级作用域中编译的；子模板里的所有内容都是在子作用域中编译的

所以，上面例子中插槽不能访问<navigation-link>组件的作用域（当然这个插槽仅仅是个文本，没有用到组件中的属性）

# 具名插槽

自 2.6.0 起有所更新。已废弃的使用 slot 特性的语法在这里。2.x版本还可使用，3.x版本不再支持

<div class="container">
<header>
  <slot name="header"></slot>
</header>
<main>
  <slot></slot>
</main>
<footer>
  <slot name="footer"></slot>
</footer>
</div>

通过name给slot添加名字，一个不带 name 的 <slot>出口会带有隐含的名字“default”。

在向具名插槽提供内容的时候，我们可以在一个 <template> 元素上使用 v-slot 指令，并以 v-slot 的参数的形式提供其名称:

<base-layout>
  <template v-slot:header>
    <h1>Here might be a page title</h1>
  </template>

  <p>A paragraph for the main content.</p>
  <p>And another one.</p>

  <template v-slot:footer>
    <p>Here's some contact info</p>
  </template>
</base-layout>

注意 v-slot 只能添加在一个 <template> 上,只有一种情况除外,把所有内容传递到一个具名slot内:

<base-layout v-slot:header>
  <p>A paragraph for the main content.</p>
  <p>And another one.</p>
</base-layout>

v-slot 也有缩写，即把参数之前的所有内容 (v-slot:) 替换为字符 #。例如 v-slot:header 可以被重写为 #header

总结：具名插槽通过name、v-slot配合使用

# 作用域插槽

上面编译作用域时说插槽的作用域和父级作用域一样，但是有时让插槽内容能够访问子组件中才有的数据是很有用的。

为了让子组件中的数据在父级的插槽内容可用，我们可以将数据作为一个 <slot> 元素的特性绑定上去:

// 假设子组件data中有user这条数据
<span>
  <slot v-bind:user="user">
    {{ user.lastName }}
  </slot>
</span>

绑定在 <slot> 元素上的特性被称为插槽 prop。现在在父级作用域中，我们可以给 v-slot 带一个值来定义我们提供的插槽 prop 的名字:

// 我们选择将包含所有插槽 prop 的对象命名为 slotProps，但你也可以使用任意你喜欢的名字
<current-user>
  <template v-slot:default="slotProps">
    {{ slotProps.user.firstName }}
  </template>
</current-user>

# 解构插槽 Prop

可以使用 ES2015 解构来传入具体的插槽 prop:

<current-user v-slot="{ user }">
  {{ user.firstName }}
</current-user>

// 重命名为person
<current-user v-slot="{ user: person }">
  {{ person.firstName }}
</current-user>

// 默认值
<current-user v-slot="{ user = { firstName: 'Guest' } }">
  {{ user.firstName }}
</current-user>

# 动态组件

有的时候，在不同组件之间进行动态切换是非常有用的，这时候就可以使用Vue内置的组件<component>加一个特殊的is特性来实现：

<!-- 组件会在 `currentComponentName` 改变时改变 -->
<component v-bind:is="currentComponentName"></component>

# keep-alive

动态组件每次改变切换组件时，Vue 都创建了一个新的 currentComponentName 实例，不但导致组件重复渲染，而且还会丢失组件状态，为了解决这个问题，我们可以用一个 <keep-alive> 元素将其动态组件包裹起来

<!-- 失活的组件将会被缓存！-->
<keep-alive>
<component v-bind:is="currentComponentName"></component>
</keep-alive>

# 异步组件

在大型应用中，我们可能需要将应用分割成小一些的代码块，并且只在需要的时候才从服务器加载一个模块

new Vue({
  // ...
  components: {
    'my-component': () => import('./my-async-component')
  }
})
